# ThingSpeak Monitor

Простое серверное веб-приложение на `Python + Flask + PostgreSQL`, разворачиваемое через `docker-compose`.

Что делает приложение:
- даёт страницу авторизации;
- после входа открывает страницу мониторинга;
- хранит список пользователей в `config.json`;
- хранит список устройств и конечных серверов в `memory.json`;
- при старте догружает из ThingSpeak уже существующие данные, если их ещё нет в базе;
- регулярно опрашивает ThingSpeak-каналы устройств;
- сохраняет новые измерения в PostgreSQL;
- отображает графики по выбранному устройству;
- даёт интерактивные графики с масштабированием, панорамированием и быстрым сбросом вида;
- отправляет новые события на конечные серверы в формате `POST JSON`.

## Структура payload для конечных серверов

```json
{
  "device_name": "sensor1",
  "device_hash": "2918084",
  "event_id": 123,
  "air_temp": 21.5,
  "air_hum": 43.2,
  "warm_stream": 112.4,
  "surface_temp": 27.1
}
```

Отправка делается асинхронно: приложение не ждёт ответа от конечного сервера в пользовательском интерфейсе и не блокирует фоновый опросчик из-за медленного получателя.

## Как запустить

1. Распакуйте архив.
2. Скопируйте `.env.example` в `.env`.
3. При необходимости измените внешний порт в `.env`:

```env
WEB_EXTERNAL_PORT=8080
```

4. При необходимости измените `config.json`.
5. Запустите:

```bash
docker compose up --build -d
```

6. Откройте браузер:

```text
http://localhost:8080/login
```

Если в `.env` указан другой порт, используйте его.

## Данные по умолчанию

### Пользователь по умолчанию

В `config.json` уже добавлен пользователь:

- `admin:admin`

Перед реальным использованием пароль лучше сменить.

### Устройство по умолчанию

В `memory.json` уже добавлено одно устройство:

- `sensor1` → `https://thingspeak.mathworks.com/channels/2918084`

## Настройка пользователей

Пользователи задаются в `config.json` списком.

Вариант с обычным паролем:

```json
{
  "username": "admin",
  "password": "admin"
}
```

Поддерживается и вариант с `password_hash`, если вы захотите хранить не открытый пароль.

## Настройка устройств и конечных серверов

Списки устройств и серверов лежат в `memory.json`.

При запуске приложение:
- читает `memory.json`;
- валидирует данные;
- показывает их в таблицах;
- при сохранении в веб-интерфейсе перезаписывает `memory.json` атомарно.

### Формат устройств

```json
{
  "name": "sensor1",
  "thingspeak_url": "https://thingspeak.mathworks.com/channels/2918084"
}
```

`device_hash` автоматически извлекается из URL канала ThingSpeak и сохраняется приложением.

### Формат конечных серверов

```json
{
  "name": "local_receiver",
  "host": "127.0.0.1",
  "port": 5050,
  "path": "/sensors/data"
}
```

Итоговый адрес отправки будет:

```text
http://127.0.0.1:5050/sensors/data
```

## Про уникальность `event_id`

В разных ThingSpeak-каналах `entry_id` могут совпадать. Поэтому в базе включена защита от дублей по паре:

- `device_hash`
- `event_id`

Это практическая и безопасная проверка уникальности для нескольких устройств.

## Историческая загрузка при старте

При запуске приложение делает стартовую догрузку данных из ThingSpeak для каждого устройства.

Что происходит:
- приложение читает историю канала через REST API `feeds.json`;
- если в базе уже есть часть записей, дубли не вставляются благодаря уникальному ограничению по `device_hash + event_id`;
- если записей много, история подгружается пакетами до 8000 записей за запрос и сдвигается назад по времени через параметр `end`.

Это позволяет после перезапуска автоматически подтянуть уже существующие данные из графиков, которых ещё нет в PostgreSQL.

## Поля ThingSpeak

Приложение читает названия и значения полей канала через API ThingSpeak и приводит их к внутренним метрикам.
Если названия полей канала нестандартные, проверьте логи сервиса `web` и при необходимости скорректируйте правила сопоставления в `app/services.py`.

## Управление графиками

На странице мониторинга доступны быстрые действия:

- колесо мыши — масштаб по времени;
- `Shift + колесо` — масштаб по оси Y;
- перетаскивание — панорамирование;
- двойной клик — сброс масштаба;
- кнопка `Y-fit` — вернуть автоматический подбор шкалы по оси Y.

## Авторизация

Авторизация простая, но доступ к `/monitoring` и ко всем `/api/*` закрыт без логина.

Что уже сделано:
- серверная проверка сессии;
- защищённые cookie-сессии Flask;
- отдельный `secret_key`;
- CSRF-проверка для изменения таблиц;
- выход из системы через POST.

Для production стоит обязательно:
- заменить `secret_key`;
- заменить пароль `admin`;
- включить HTTPS;
- установить `session_cookie_secure=true` в `config.json`.
